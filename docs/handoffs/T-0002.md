### Exact commands

```sh
# 1. Start isolated test database
docker compose -f docker-compose.test.yml up -d

# 2. Wait for health
until docker compose -f docker-compose.test.yml exec -T postgres-test pg_isready -U sidus_test -d sidus_test; do sleep 1; done

# 3. Run Go migration using Docker Go toolchain (no host Go required)
docker run --rm --network sidus-test_default -v "$(pwd)/services/core:/app" -w /app \
  -e DATABASE_URL="postgres://sidus_test:sidus_test_password@postgres-test:5432/sidus_test?sslmode=disable" \
  golang:1.22-alpine go run ./cmd/migrate

# 4. Run integration tests with TEST_DATABASE_URL
docker run --rm --network sidus-test_default -v "$(pwd)/services/core:/app" -w /app \
  -e TEST_DATABASE_URL="postgres://sidus_test:sidus_test_password@postgres-test:5432/sidus_test?sslmode=disable" \
  golang:1.22-alpine go test ./internal/contentsource/... -run Integration -v

# 5. Destroy only test project resources (dev postgres/volume untouched)
docker compose -f docker-compose.test.yml down -v
```

`//d/Sidus/services/core` style host paths to stop MSYS from mangling the `-v`/`-w` paths.)

## Review fix-up (earlier pass)

Two review findings on the original delivery, both fixed:

1. **Audit accuracy.** `Update` previously recorded every *supplied* field name as
   "changed," even if the value matched what was already stored. Fixed: `Update` now fetches
   the current row, diffs each supplied value against it, applies/records only fields that
   actually differ, and returns `400 no_changes` (no write, no event, `updated_at`
   untouched) when supplied fields are all identical to the stored values. A request with no
   updatable field at all still returns `400 no_updatable_fields`.
2. **Integration test data safety.** The integration tests wrote to
   `content_source_events`/`content_source_reviews` (immutable — DB trigger rejects
   UPDATE/DELETE) and then attempted a `DELETE` cleanup with the error silently swallowed,
   leaving orphaned rows in whatever database `TEST_DATABASE_URL` pointed at. Fixed: removed
   the cleanup attempts, added `docker-compose.test.yml` defining a disposable
   `postgres-test` service (tmpfs-backed, separate network/volume from the dev `postgres`
   service), and documented in `postgres_store_test.go` that `TEST_DATABASE_URL` must always
   point at that disposable instance — never dev/prod — and that it should be destroyed
   after the run.

## Goal

Let human curators complete metadata on `pending` content sources (including the seeded
0610/5090 syllabus rows) without bypassing rights approval, and record every change in an
append-only, DB-immutable audit trail.

## Delivered

- **`PATCH /content-sources/{id}`** (`services/core/internal/contentsource/handlers.go`):
  - `actorId` required in body (`400 missing_required_fields` if absent/blank).
  - Updatable fields: `title`, `owner`, `sourceUrl`, `sourceHash`, `licenceReference`,
    `permittedUse`, `allowedAudience`, `syllabusCode`.
  - Any supplied field that is empty/whitespace-only → `400 blank_fields`.
  - No updatable field supplied at all → `400 no_updatable_fields`.
  - Updatable fields supplied but every one matches the current stored value →
    `400 no_changes` (no write, no event, `updated_at` untouched).
  - `syllabusCode` not in `{0610,5090}` → `400 invalid_syllabus_code`.
  - `sourceUrl` not an absolute http/https URL → `400 invalid_source_url`.
  - Duplicate `sourceUrl` → `409 duplicate_source_url`.
  - Source not `pending` → `409 invalid_status_transition`.
  - Missing source → `404 not_found`.
  - Success → `200` with the updated source; **status stays `pending`** (PATCH never approves).
- **Audit table** `content_source_events` (migration `0004_create_content_source_events.sql`):
  append-only; columns `content_source_id`, `event_type` (`CHECK` = `metadata_updated`),
  `actor_id`, `event_time`, `changed_fields TEXT[]`. Immutability enforced by a
  `BEFORE UPDATE OR DELETE` trigger that raises. `content_source_reviews` untouched.
  - Records **field names only** — never PDF content, extracted text, diagrams, or the
    previous/new field values.
  - One event per successful update; `changed_fields` = the field names whose stored value
    actually changed (real value-diff against the currently stored row — see decisions).
- **Store** (`store.go`, `postgres_store.go`): `Update(ctx, id, UpdateInput)`
  `(Source, changed []string, err error)`. Runs in a transaction: `SELECT ... FOR UPDATE`,
  rejects non-pending, diffs each supplied field against the locked row, builds dynamic `SET`
  from only the fields that differ, bumps `updated_at`, inserts the event — all skipped if
  nothing actually differs. Sentinels: `ErrNoUpdatableFields` (nothing supplied),
  `ErrNoChanges` (supplied but identical to stored).
- **Model** (`model.go`): `Event`, `EventType`/`EventMetadataUpdated`, `UpdateInput`,
  `UpdatableFields`.
- **Shared contracts** (`packages/shared/src/contracts.ts`): `UpdateContentSourceRequest`,
  `ContentSourceEvent`, `ContentSourceEventType`, `UPDATABLE_CONTENT_SOURCE_FIELDS`.
- **Python parity** (`services/ai/app/content_sources.py`): `ContentSourceEventType` enum +
  `ContentSourceEvent` model, mirroring the Go/TS contract. Ingestion gate unchanged.
- **`docker-compose.test.yml`** (new, repo root): disposable `postgres-test` service (tmpfs
  data dir, distinct port `55432` by default) for running the immutable-audit integration
  tests without ever touching the dev/prod database.

## Decisions / assumptions

- **`changed_fields` = a real value-diff against the currently stored row** (superseded from
  the original "fields supplied" rule per review finding 1: recording untouched fields as
  "changed" made the audit trail inaccurate). `Update` locks the row (`SELECT ... FOR
  UPDATE`), compares each supplied field's value to what's already stored, and only
  applies/records fields that differ. The value itself is still never stored — only the
  field name — so this is a per-request diff computed in memory, not a persisted value
  comparison. A request with ≥1 supplied field but zero actual differences is rejected with
  `400 no_changes` before any write, keeping `changed_fields` on a written event always
  non-empty.
- **Updatable fields are optional JSON pointers.** Absent and explicit `null` both mean "no
  change"; a present `""`/whitespace value is a validation error (`400`), never a "clear the
  field" — rights fields must not be blanked via PATCH.
- **PATCH never changes `status`.** Approval stays a separate endpoint requiring all existing
  approval fields. The seeded 0610/5090 rows stay `pending` until a human supplies verified
  rights metadata (via PATCH) and separately approves. No owner/hash/licence/audience data
  invented.
- **Duplicate URL** relies on the existing `content_sources.source_url` UNIQUE constraint
  (`23505` → `ErrDuplicateSourceURL` → `409`), consistent with `POST`.
- **URL validation**: `net/url.Parse` + scheme ∈ {http,https} + non-empty host (rejects
  `not-a-url`, `ftp://…`, scheme-relative `//host`, bare `host/path`).
- **Python event model** added for contract parity only; the ingestion gate does not consume
  events (YAGNI-aware, but the task explicitly asks to keep Go/Python contracts aligned).
- **Integration tests require a disposable database** (review finding 2): the tests write
  immutable `content_source_reviews`/`content_source_events` rows and cannot clean up (DB
  trigger rejects UPDATE/DELETE; the earlier silent-fail DELETE attempts were removed).
  `docker-compose.test.yml` defines `postgres-test`, a tmpfs-backed disposable service on a
  separate port/network entry from the dev `postgres` service in `docker-compose.yml`.
  `TEST_DATABASE_URL` must point at it; the CI/local flow is: `up -d` → migrate → test →
  `down -v`.
