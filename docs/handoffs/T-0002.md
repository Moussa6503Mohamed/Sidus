# T-0002 handoff

**Status:** review
**Owner:** Claude Code agent
**Depends on:** T-0001 (done)

## Goal

Let human curators complete metadata on `pending` content sources (including the seeded
0610/5090 syllabus rows) without bypassing rights approval, and record every change in an
append-only, DB-immutable audit trail.

## Delivered

- **`PATCH /content-sources/{id}`** (`services/core/internal/contentsource/handlers.go`):
  - `actorId` required in body (`400 missing_required_fields` if absent/blank).
  - Updatable fields: `title`, `owner`, `sourceUrl`, `sourceHash`, `licenceReference`,
    `permittedUse`, `allowedAudience`, `syllabusCode`.
  - Any supplied field that is empty/whitespace-only → `400 blank_fields`.
  - No updatable field supplied → `400 no_updatable_fields`.
  - `syllabusCode` not in `{0610,5090}` → `400 invalid_syllabus_code`.
  - `sourceUrl` not an absolute http/https URL → `400 invalid_source_url`.
  - Duplicate `sourceUrl` → `409 duplicate_source_url`.
  - Source not `pending` → `409 invalid_status_transition`.
  - Missing source → `404 not_found`.
  - Success → `200` with the updated source; **status stays `pending`** (PATCH never approves).
- **Audit table** `content_source_events` (migration `0004_create_content_source_events.sql`):
  append-only; columns `content_source_id`, `event_type` (`CHECK` = `metadata_updated`),
  `actor_id`, `event_time`, `changed_fields TEXT[]`. Immutability enforced by a
  `BEFORE UPDATE OR DELETE` trigger that raises. `content_source_reviews` untouched.
  - Records **field names only** — never PDF content, extracted text, diagrams, or the
    previous/new field values.
  - One event per successful update; `changed_fields` = the field names supplied and applied
    (see decisions).
- **Store** (`store.go`, `postgres_store.go`): new `Update(ctx, id, UpdateInput)`
  `(Source, changed []string, err error)`. Runs in a transaction: `SELECT ... FOR UPDATE`,
  rejects non-pending, builds dynamic `SET` from supplied fields, bumps `updated_at`,
  inserts the event. New sentinel `ErrNoUpdatableFields`.
- **Model** (`model.go`): `Event`, `EventType`/`EventMetadataUpdated`, `UpdateInput`,
  `UpdatableFields`.
- **Shared contracts** (`packages/shared/src/contracts.ts`): `UpdateContentSourceRequest`,
  `ContentSourceEvent`, `ContentSourceEventType`, `UPDATABLE_CONTENT_SOURCE_FIELDS`.
- **Python parity** (`services/ai/app/content_sources.py`): `ContentSourceEventType` enum +
  `ContentSourceEvent` model, mirroring the Go/TS contract. Ingestion gate unchanged.

## Decisions / assumptions

- **`changed_fields` = the fields supplied in the request that passed validation**, not a
  value-level diff. Because field values are never stored, a value-diff would be unverifiable
  from the trail; recording which fields the curator addressed is the meaningful, deterministic
  audit signal and guarantees a non-empty array (≥1 field is required).
- **Updatable fields are optional JSON pointers.** Absent and explicit `null` both mean "no
  change"; a present `""`/whitespace value is a validation error (`400`), never a "clear the
  field" — rights fields must not be blanked via PATCH.
- **PATCH never changes `status`.** Approval stays a separate endpoint requiring all existing
  approval fields. The seeded 0610/5090 rows stay `pending` until a human supplies verified
  rights metadata (via PATCH) and separately approves. No owner/hash/licence/audience data
  invented.
- **Duplicate URL** relies on the existing `content_sources.source_url` UNIQUE constraint
  (`23505` → `ErrDuplicateSourceURL` → `409`), consistent with `POST`.
- **URL validation**: `net/url.Parse` + scheme ∈ {http,https} + non-empty host (rejects
  `not-a-url`, `ftp://…`, scheme-relative `//host`, bare `host/path`).
- **Python event model** added for contract parity only; the ingestion gate does not consume
  events (YAGNI-aware, but the task explicitly asks to keep Go/Python contracts aligned).
