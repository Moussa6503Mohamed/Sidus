## Decisions / assumptions

- No local Go toolchain on this machine; all Go tooling run via `golang:1.22-alpine` in Docker. Documented so future agents know why `go.sum` exists without a host `go`.
- `owner`, `source_hash`, `licence_reference`, `allowed_audience` are nullable on `content_sources` and only enforced (non-null) at approval time, not at creation — this keeps `POST /content-sources` usable for lightweight "here's a link, rights TBD" entries while still blocking approval per the acceptance checks.
- No update/patch endpoint exists (out of the "create/get/list/approve/reject" scope given). Rights fields not supplied at creation cannot currently be added later — flagged as an open question/follow-up in `docs/tasks/active.md`.
- Seed rows for CAM-0610-2026 / CAM-5090-2026 leave `owner`/`source_hash`/`licence_reference`/`allowed_audience` null rather than guessing values not present in `docs/content-provenance-register.md`. `permitted_use` is copied verbatim from the register's documented "Approved use" text. These seeds cannot pass approval until a reviewer supplies the missing fields — this is intended behavior, not a bug.
- `decisionDate` is accepted as an optional RFC3339 string on approve/reject requests and defaults to the server's current time if omitted, so a decision date is always recorded even if the caller doesn't supply one.
- Content-source review immutability is enforced at the database level (trigger blocking `UPDATE`/`DELETE`), not just in application code, so it holds even against direct SQL access.
- `expired` is included in the schema/type CHECK constraints and shared contracts for forward-compatibility, but no endpoint transitions a source to `expired` in this task (not listed in the Core API scope).
