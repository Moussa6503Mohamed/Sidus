## Goal

Clerk owns authentication; Sidus Core owns authorization. No user-controlled
`actorId`/`reviewerId`. Audit identity (event actor, review reviewer) derives only from the
verified Clerk session subject. Role authorization from the verified `sidus_role` claim.

## Decisions / assumptions

- **Clerk Go SDK pinned to v2.5.0.** v2.6.0/v2.7.0 require Go ≥1.24; the repo (and the
  `golang:1.22-alpine` toolchain used everywhere) is Go 1.22. v2.5.0 has the identical
  verification API (`jwt.Decode`/`GetJSONWebKey`/`Verify`, `AuthorizedPartyHandler`,
  `CustomClaimsConstructor`, `SessionClaims.Custom`). Revisit if/when the toolchain moves to
  ≥1.24.
- **Issuer pinned explicitly** in addition to the SDK's built-in Clerk-issuer format check,
  so tokens from any other Clerk instance are rejected.
- **JWK TTL cache** (default 1h) balances "no Backend API call per request" against picking up
  key rotation; only an unknown/expired-cache `kid` triggers a fetch.
- **Read permission granted to editor/reviewer/admin, denied to learner** — "learner: no
  content-source access" means learner cannot even list/get sources.
- **Fail closed:** content-source routes never mount without a verifier, so a
  misconfiguration cannot expose them unauthenticated.
- **`actorId`/`reviewerId` removed from request bodies.** The obsolete
  `TestUpdate_MissingActorID_Returns400` was replaced by token-based auth tests; identity now
  comes only from the verified subject (`TestUpdate_ActorFromVerifiedSubject`,
  `TestReject_ReviewerFromVerifiedSubject`).
- **Next.js 16 `proxy.ts`** is the current middleware entrypoint; `<Show when=...>` replaced
  `SignedIn`/`SignedOut` in Clerk v7.
- **`apps/web/.env.local` dummy keys** are needed only so `next build`/typecheck run offline;
  they are gitignored and non-functional. Replace with real keys for a working sign-in flow.

## Manual Clerk Dashboard steps (human, before beta)

1. Create the Clerk application; put real `pk_...`/`sk_...` in gitignored `.env.local` files
   (never in `.env.example`).
2. Sessions → Customize session token → add claim **`sidus_role`** =
   `{{user.public_metadata.sidus_role}}`.
3. Store each user's role in **public metadata** `sidus_role` ∈ {learner, editor, reviewer,
   admin}. **Manually set the first admin** (`{"sidus_role":"admin"}`).
4. Configure production domain/origins; set Core `CLERK_JWT_ISSUER` and
   `CLERK_AUTHORIZED_PARTIES` (comma-separated prod origins), and AI `CLERK_JWKS_URL`.

Full detail in `docs/auth-setup.md`.
